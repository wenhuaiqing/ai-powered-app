# Screenshots checklist

The repo's main README intentionally ships without images so the text
lands on its own first. When you want to add visuals (CV link, recruiter
share, etc.), drop PNGs / WEBPs here named exactly as below and add a
`Screenshots` section to the root `README.md` referencing them.

| File | What to capture | Where |
|---|---|---|
| `dashboard.png`     | Dashboard at default zoom — hero strip, KPI cards, 7 agent cards, sample prompts hint. | `/dashboard` |
| `orb-trace.png`     | Orb panel mid-stream with at least one tool call visible. Multi-agent run (stamp duty + matcher) is the most impressive. | Click sample prompt #4 from the dashboard |
| `answer-card.png`   | Final answer card with citations + agent badges. Best after a Compliance question. | After prompt #1 |
| `valuations.png`    | Predictor result panel — predicted price, 80% CI, contributions bar chart. | `/valuations` |
| `compliance.png`    | Compliance Hub with cited results + relevance meter strength bars. | `/compliance` after "bond limits" |
| `insights.png`      | Market Insights — all 4 charts visible + Leaflet bubble map. | `/insights` |
| `mobile-rai.png`    | Mobile home page (Rai full-page). | Click "Mobile view" toggle in header |
| `mobile-pipeline.png` | Pipeline list as cards with the docked Rai bar at the bottom. | Mobile view → `/pipeline` |

Tips:
- Take at 1440 × 900 desktop and 390 × 844 mobile (iPhone 14 logical size).
- Use a fresh / cleared chat (`New chat` pill) so the orb shows the sample prompts when relevant.
- Crop tightly — recruiter shares look better without browser chrome.
